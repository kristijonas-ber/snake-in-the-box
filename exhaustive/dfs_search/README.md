# dfs_search — exhaustive DFS search for the longest snake

## Quick reference

| Run | Finds |
|---|---|
| `mpirun -n <procs> ./dfs_search` | All longest snakes in dimension `N` (default 6) |
| `mpirun -n <procs> ./dfs_search_replay` | Same result, faster when prefix generation (not search) is the bottleneck |
| `./dfs_search --decode <file>.bin` | Prints the snakes stored in a `job_outputs/` result file |

```bash
make                                     # build both binaries, defaults N=6
mpirun --oversubscribe -n 5 ./dfs_search # → 26 edges, count 1
```

## What it does

A canonical-augmentation depth-first search — a symmetry-reduction rule that
only extends paths already in canonical form, skipping symmetric duplicates —
that provably finds **every** longest snake in `Q_N`. It splits the search space into *prefixes* — generated by a
deterministic, streaming walk so no table of prefixes ever needs to fit in RAM —
and dispatches each prefix to a worker rank over MPI. Two passes: the first finds
the longest length `L`, the second re-walks only the prefixes that hit `L` and
emits every snake of that length. `dfs_search` load-balances prefixes
dynamically (master/worker); `dfs_search_replay` skips the dispatcher and has
every rank regenerate and statically own its share of prefixes — pick whichever
one generation cost favors (see file header comments for the trade-off).

## Parameters

| `#define` | default | meaning |
|---|---|---|
| `N` | 6 | hypercube dimension to search |
| `PREFIX_LENGTH` | 11 | depth of the prefix = scheduling granularity |
| `SLICE_COUNT` / `SLICE_ID` | 1 / 0 | split the search into `SLICE_COUNT` independent runs (e.g. across machines); this run computes slice `SLICE_ID` |

Override at build time, e.g. `make DEFS="-DN=8 -DPREFIX_LENGTH=18"`. More
advanced knobs (window mode, checkpoint/resume, Knuth probes) are documented in
`config.hpp`.

## Output

Results are written to `../job_outputs/`: a text summary, per-rank fragments,
and `.bin` files holding the longest snakes found — read them back with
`--decode`.

## Example output

`--decode` prints one line per snake: the vertex sequence, built by XOR-ing
each transition (bit position flipped) into the previous vertex, starting
from `0`. Worked by hand for `N=2` (max snake length is 2 edges), transitions
`0 1` decode to:

```
0 1 3
```
(vertex `0` → flip bit 0 → `1` → flip bit 1 → `3`, a valid induced path in `Q_2`)
